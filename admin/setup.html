首次启用

系统主体已经完成。以下步骤只需由仓库所有者执行一次，之后部门成员使用自己的 GitHub 账号登录。

  1. 在本机运行仓库中的 cms-auth/setup.ps1，按提示登录 Cloudflare。
  2. 助手会显示登录服务地址，并打开 GitHub OAuth App 创建页面。
  3. 把助手给出的主页与回调地址填入 GitHub，然后在本机输入 Client ID 和 Client Secret。
  4. 助手会生成签名密钥、部署服务并更新后台配置；最后提交并部署该配置即可。

不要把 Client Secret 发到聊天、邮件或提交到 GitHub。它只应通过初始化助手写入 Cloudflare 的加密 Secret。

需要使用后台的成员还必须被添加为 LoganCome/robailab.github.io 的 GitHub 协作者。移除协作者会同步撤销其写入权限。
